## Technical Insights

### 5 Patterns Identified

1. **`light-on-off`** — "turn on/off [area] lights" — works for exact area names
2. **`climate-set-temp`** — "set [area] temperature to N" — works for thermostat entities
3. **`climate-comfort`** — "make it warmer/cooler [in area]" — relative adjustments, no absolute values
4. **`scene-activate`** — "activate [scene name]" — exact scene matching
5. **`climate-get-status`** — Query patterns for temperature/humidity — read-only, non-destructive

### Broken Scenarios

- **Climate control:** No fan (`turn on the fan`), no HVAC mode (`set to cool mode`), no timer (`turn off in 5 minutes`)
- **Entity resolution:** "front room" ≠ "Front Room" (area), "Keller" (German) becomes "Keller" (untranslatable)
- **Colloquial names:** Patterns expect exact HA entity/area names, fail on aliases
- **Multi-room:** No "all lights" or "downstairs" group patterns
- **Color commands:** No pattern for "make it blue", falls to broken LLM parameter extraction

### Accidental Bail Mechanism

Temporal commands like "turn off in 5 minutes" generate leftover tokens (`in`, `5`, `minutes`) that drop confidence below threshold (0.8), causing automatic fallback to LLM. This is not intentional design but a side effect of the confidence penalty. If leftover tokens were weighted differently, temporal commands would incorrectly match the pattern.

## Architectural Issues

1. **Pattern-skill binding is brittle:** Direct switch-dispatch in `DirectSkillExecutor` — adding a pattern requires code change
2. **Confidence is conflated with pattern quality and resolution quality:** A high-confidence pattern match can still fail at skill execution
3. **Entity resolution has no data source validation:** If entity location cache is stale or missing, resolution is blind
4. **No STT quality gate:** Garbled input like "You're not doing anything..." (background audio bleed) confidently routes to wrong agents
5. **Multi-turn state is not persistent:** Conversation context lost between turns, breaking handoff from fast-path to orchestrator
