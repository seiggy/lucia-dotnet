## Design Decisions

1. **Normalize in validator AND YAML** — Belt and suspenders approach ensures resilience to either naming convention
2. **Assertion helpers on base class** — Reuses existing validator infrastructure, avoids splitting logic
3. **Failure classification heuristic** — Checks NoToolCall → Hallucination → WrongTool → WrongEntity → WrongParams to ensure most specific diagnosis wins
4. **Trace JSON with metadata** — Includes source (eval run ID, execution index), model, category, failure type, and raw trace for future analysis
5. **Filtered MemberData accessors** — Enables regression suites and model-specific test runs without test code changes
